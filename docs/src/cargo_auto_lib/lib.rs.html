39//! ## Try it
40//!
41//! First, install the tool for task automation in Rust projects:
42//!
43//! ```bash
44//! cargo install cargo-auto
45//! # Generate a new Rust CLI project:
46//! cargo auto new_cli hello_world
47//! # Open the `hello_world` project in VSCode:
48//! code hello_world
49//! # Open the generated directory `automation_tasks_rs` as an independent rust project in VSCode.
50//! code hello_world/automation_tasks_rs
51//! ```
52//!
53//! Now we can analyze the automation code. There is already this dependency inside `Cargo.toml` for our library:  
54//!
55//! ```toml
56//! cargo_auto_lib="2.0.2"
57//! ```
58//!
59//! Review the code in `automation_tasks_rs/main.rs`. The `cl::` namespace is the alias for `cargo_auto_lib`.  
60//! Example:  
61//!
62//! ```rust ignore
63//! /// cargo build --release
64//! fn task_release() {
65//!     let cargo_toml = CargoToml::read();
66//!     cl::auto_version_increment_semver_or_date();
67//!     cl::auto_cargo_toml_to_md();
68//!     cl::auto_lines_of_code("");
69//!
70//!     cl::run_shell_command("cargo fmt");
71//!     cl::run_shell_command("cargo build --release");
72//!     println!(
73//!         r#"
74//!   {YELLOW}After `cargo auto release`, run the compiled binary, examples and/or tests{RESET}
75//! {GREEN}./target/release/{package_name} arg_1{RESET}
76//!   {YELLOW}if ok then{RESET}
77//! {GREEN}cargo auto doc{RESET}
78//! "#,
79//! package_name = cargo_toml.package_name(),
80//!     );
81//! }
82//! ```
83//!
84//! You can see this function will increment the version in `Cargo.toml`.  
85//! Then it will copy some data from `Cargo.toml` to README.md (title, description, version, author,...).  
86//! It will count the lines of code and create badges in README.md.  
87//! Then comes the traditional Rust part: cargo fmt and cargo build --release.  
88//! Finally, it will show on the screen the instructions on how to continue developing.  
89//!
90//! Run (in your main rust project):
91//!
92//! ```bash
93//! cargo auto release
94//! ```
95//!
96//! Now open the README.md and you will see the data that this automation task copied from other places. Therefore you change this data only in one place, the automation task copies them wherever needed.
